BI enables fast access of data by

(A) Providing data connectors

(B) ** Aggregating data

(C) Using metadata

(D) All of the above

EXPLANATIONS BELOW

Concept note-1: -Aggregations in Power BI can improve query performance over very large DirectQuery datasets. By using aggregations, you cache data at the aggregated level in-memory.

Concept note-2: -Data aggregation is one of the most powerful methods of compiling data for statistical analysis. This process can help you extract data that provides valuable insights into the effectiveness of your products, services, or marketing campaigns.

Concept note-3: -The aggregated table is a table in Power BI aggregated by one or more fields from the DirectQuery source table. In our case, the Aggregated table would be grouped by specific fields from the FactInternetSales table. You can use various methods to create the aggregated table.

Concept note-4: -For example, raw data can be aggregated over a given time period to provide statistics such as average, minimum, maximum, sum, and count. After the data is aggregated and written to a view or report, you can analyze the aggregated data to gain insights about particular resources or resource groups.

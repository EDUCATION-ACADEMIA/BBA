How does the SAP HANA architecture improve SAP S/4HANA system performance? Please choose the correct answer

(A) SAP HANA allows you to build aggregate tables on top of aggregates and, in addition, special versions of the database tables to support special applications.

(B) ** SAP HANA organizes data in line item tables and can aggregate data from these line item tables at runtime

(C) SAP HANA makes extensive use of database indexes providing common access paths to data in order to improve access speed

(D) SAP HANA uses a structured hierarchical data model with a multitude of lean prebuilt aggregated tables to write efficient code

EXPLANATIONS BELOW

Concept note-1: -S/4 HANA applications leverage speed, context and data accessibility like never before. Improved performance and speed: S/4HANA enables improved performance by allowing you to plan, execute, generate reports and analytics based on live data, as well as triggering period closing and providing improved predictions.

Concept note-2: -There are three key features that allow the HANA platform to solve the problems ERP and BI were facing, these are: In-memory computing; Columnar database managemnet & data compression; Parallel processing.

Concept note-3: -S/4HANA provides light process automation so organizations can automate routine tasks such as running reports that previously were done manually, freeing up workers to focus on more productive actions. Robotic process automation tools can also be added to speed and streamline business processes.

Concept note-4: -An SAP HANA system is composed of three main components: the host, the system, and the instance. A host is the operating environment in which the SAP HANA database runs. The host provides all the resources and services (CPU, memory, network, and operating system) that the SAP HANA database requires.

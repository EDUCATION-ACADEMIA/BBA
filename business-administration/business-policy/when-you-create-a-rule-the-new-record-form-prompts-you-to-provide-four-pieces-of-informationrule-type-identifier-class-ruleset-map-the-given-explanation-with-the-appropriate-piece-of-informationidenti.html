Concept note-1: -Pega Platform creates a unique instance key-stored by using the property . pzInsKey-to identify each rule on the system. The instance key generally consists of four elements separated by a space: The internal name for the rule type.

Concept note-2: -Developers create a new version of a rule whenever case behavior needs to change. Pega Platform maintains a history of changes to a rule, allowing developers to review the change history and undo changes if needed.

Concept note-3: -Create a flow rule from the App Explorer and build the process. Create a flow rule from the Create menu and build the process.

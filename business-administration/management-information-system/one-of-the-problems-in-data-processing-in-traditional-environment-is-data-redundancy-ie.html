[SOURCES]
one of the problems in data processing in traditional environment is data redundancy i.e.,

(A) ** the presence of duplicate data

(B) data inconsistency

(C) data loss

(D) all wrong

EXPLANATIONS BELOW

Concept note-1: -Over time, this traditional file management environment creates problems such as data redundancy and inconsistency, program-data dependence, inflexibility, poor security, and lack of data sharing and availability.

Concept note-2: -Data redundancy: duplicate data in multiple files, leading to data inconsistency, different values used for the same attribute.

Concept note-3: -Duplication of data is called data redundancy. Duplication of data should be checked always as data redundancy takes up the free space available in the computer memory. Data redundancy occurs when the same piece of data is stored in two or more separate places and is a common occurrence.

Concept note-4: -Redundancy causes insertion, deletion, and updation anomalies. Redundancy can be avoided by normalizing the database, maintaining master data, etc.
